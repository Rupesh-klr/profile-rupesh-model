## 🔗 How your answers map to the website

| Section | Goes into | JSON field |
|---------|-----------|-----------|
| A (About) | Hero + About | `person`, `sections[about]` |
| B (Contact) | Contact + footer | `sections[contact]`, `person.social` |
| C (Social) | Header + footer + Contact | `person.social` |
| D (Numbers) | Stats band | `sections[stats]` |
| E (Audience) | Audience / Challenges | `sections[help/challenges]` |
| F (Testimonials) | Stories | `sections[stories]` |
| G (Business model) | Offerings + CTA | `sections[offerings]`, `ui.cta` |
| H (Achievements) | About credentials | `sections[about].credentials` |
| I (Mission) | Mission card | `sections[about].mission` |
| J (Vision/Future) | New "Vision" block (we can add this section) | new `sections[vision]` |
| K (Work on / team) | Internal planning (not shown publicly) | — |
| L (Ideas / brand) | Palette + tone | `?p=` palette, fonts, colours |

> When the answers come back, we plug them into the JSON files in `src/data/` and the site updates
> automatically. Sections J (Vision) and a Team/Hiring block can be added as new section types if you
> want them shown publicly.
